Answer (Bonus) : principes SOLID :

Principe de responsabilité unique (SRP) : chaque classe a une seule responsabilité claire (ex : GestionnaireEmploiDuTemps gère  l’emploi du temps et la notification).

Principe ouvert/fermé (OCP): le système est ouvert à l’extension (via décorateurs, observateurs) mais fermé à la modification.

Principe de substitution de Liskov (LSP): les décorateurs et observateurs substituent correctement leurs interfaces.

Principe de ségrégation des interfaces (ISP): les interfaces sont fines et ciblées.

Principe d’inversion des dépendances (DIP): le code dépend d’abstractions (interfaces ICours, Observer) et non de classes concrètes.
